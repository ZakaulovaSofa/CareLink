# Архитектура: платформизация

## Функциональность, заменяемая платформами:

В ИС можно заменить **сервис уведомлений** на внешнюю платформу (например, OneSignal).

Эта платформа предоставляют готовое решение для отправки push-, email- и in-app уведомлений, что позволяет избежать разработки собственной системы.

Интеграция осуществляется через API: сервис заявок публикует событие, после чего backend вызывает API OneSignal для отправки уведомления пользователю.

При реализации MVP и на первых стадиях развития системы OneSignal по тарифам free/growth покроет все основные требования к уведомлениям. При дальнейшем росте системы и усложнении бизнес-логики возможно постепенное вынесение уведомлений в отдельный внутренний сервис.

Также в ИС в рамках 2 версии продукта можно заменить функциональность **формирования отчётов для модераторов** на внешнюю BI-платформу (например, Microsoft Power BI).

Это решение подключается к БД и строит отчёты и дашборды, убирая необходимость в разработке собственной аналитической системы.

Power BI может использоваться на всех этапах развития системы — от MVP до зрелого продукта. На начальном этапе достаточно тарифа Pro для ограниченного числа модераторов. При росте системы возможен переход на Premium-тарифы.

## Стратегия платформизации ИС

Во второй версии продукта система расширяется за счёт внедрения обучающих курсов и роли контент-мейкера. Это позволяет трансформировать приложение из инструмента координации волонтёров в **платформу с образовательным компонентом**.

Платформизация строится вокруг взаимодействия двух типов пользователей:

-   волонтёров (потребителей контента)

-   контент-мейкеров (создателей курсов)

## Бизнес-модель

Основная модель: **B2C → C2C**

-   **B2C** — волонтёры покупают курсы, которые созданы платформы

-   **C2C** — контент-мейкеры продают курсы волонтёрам через платформу

На старте платформы целесообразно использовать B2C-подход: платформа самостоятельно или с привлечением экспертов формирует базовый набор обучающих курсов. Это позволяет решить проблему отсутствия контента и привлечь первых пользователей.

По мере роста платформы реализуется C2C-модель: пользователи (контент-мейкеры) получают возможность публиковать собственные курсы и продавать их другим пользователям. Платформа в этом случае выступает посредником и получает комиссию.

Таким образом, B2C используется как стратегия запуска, а C2C — как стратегия масштабирования.

Несмотря на то, что основная функциональность системы (координация волонтёров) остаётся бесплатной, добавление marketplace курсов:

-   создаёт источник дохода

-   привлекает новых пользователей

-   повышает вовлечённость
